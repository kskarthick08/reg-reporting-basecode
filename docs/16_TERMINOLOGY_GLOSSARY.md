# Terminology Glossary

- `Workflow`
  - A governed unit of work that moves through BA, DEV, REVIEWER, and COMPLETED stages.

- `Gap Analysis`
  - BA-stage analysis that maps regulatory requirements to model fields and classifies match status.

- `Functional Specification`
  - Persisted BA deliverable derived from gap run output; downloadable in JSON or CSV.

- `Stage Exit Gate`
  - Readiness check used before stage submission.

- `Send-Back`
  - Controlled backward transition with required reason code and detail.

- `Submission XML`
  - XML artifact linked/generated for reviewer validation.

- `Analysis Run`
  - Persisted execution record for BA/DEV/REVIEWER actions and outputs.

- `Artifact`
  - Uploaded or generated file with metadata and optional extracted content.

- `RAG Chunk`
  - Indexed text fragment used for contextual retrieval during analysis/chat.
